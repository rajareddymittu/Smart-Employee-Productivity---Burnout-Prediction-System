# 5. System Development Life Cycle (SDLC)

## 5.1 Introduction

The System Development Life Cycle (SDLC) is a structured methodology used to plan, design, develop, test, deploy, and maintain software systems. It ensures that software is developed systematically while meeting user requirements, maintaining quality, and reducing project risks.

For the **Smart Employee Productivity & Burnout Prediction System Using Machine Learning**, the SDLC provides a disciplined approach for managing the entire software development process.

---

# 5.2 SDLC Model Selected

The project follows the **Agile Software Development Model**.

Agile enables the project to be developed incrementally, allowing continuous testing, regular feedback, and iterative improvements. Since the project contains multiple modules such as authentication, employee management, dashboards, and machine learning, Agile provides flexibility to develop and validate each module independently.

### Reasons for Selecting Agile

- Incremental development
- Continuous testing
- Faster feedback
- Easy requirement changes
- Better collaboration
- Reduced project risk

# 5.5 Agile vs Waterfall

| Feature | Agile | Waterfall |
|---------|--------|-----------|
| Development | Iterative | Sequential |
| Customer Feedback | Continuous | Limited |
| Requirement Changes | Easy | Difficult |
| Testing | Continuous | End of project |
| Delivery | Incremental | Final release |
